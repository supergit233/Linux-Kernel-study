# 内核移植调试与工程问题问答

## 学习目标

- 把启动链常见故障现象映射回 `U-Boot`、内核、根文件系统三层
- 建立一套适合板级移植的固定排查顺序
- 形成面向工程落地的问题问答清单

## 导读

### 本章定位

这一章是“内核移植学习项目”的工程问答章，负责把启动链问题重新组织成可直接落地的排查入口。

### 核心对象

- `U-Boot`
  - 镜像加载、DDR、启动参数
- Linux 内核
  - 设备树、内存、中断、驱动模型
- 根文件系统
  - 挂载路径、`init`、用户态基础环境

### 关键函数

- `bootm / bootz`
- `start_kernel()`
- `panic()`
- `prepare_namespace()`
- `mount_root()`
- `run_init_process()`

### 主流程

现象 -> 判断停在哪一层 -> 找到对应入口函数 -> 再回到板级配置、镜像、参数和驱动。

## 1. 常见问题分层

### 1.1 串口完全没输出

优先看：

- `U-Boot` 早期串口
- pinmux
- 时钟
- 串口基址

### 1.2 `U-Boot` 正常，但跳内核后无输出

优先看：

- 内核镜像地址
- `dtb` 地址
- `bootargs` 中 `console=`
- 内核解压入口是否正确

### 1.3 内核有输出，但 panic 在早期

优先看：

- 内存布局
- 设备树内存节点
- 中断控制器
- 定时器

### 1.4 内核起来，但挂载根文件系统失败

优先看：

- `root=`
- `rootfstype=`
- 对应存储驱动
- 设备节点命名

### 1.5 进入 rootfs 后找不到 `init`

优先看：

- `/sbin/init`
- `init=`
- 动态库
- 根文件系统内容完整性

## 2. 固定排查顺序

1. 串口是否通
2. DDR 是否稳
3. `U-Boot` 是否能读镜像
4. 内核日志是否出现
5. 设备树是否匹配当前板子
6. 存储驱动是否起来
7. `rootfs` 是否可挂载
8. `init` 是否存在并可执行

## 3. 一句话总结

内核移植问题的关键不是立刻深入某一层源码，而是先判断启动链到底停在 `U-Boot`、内核还是根文件系统。
