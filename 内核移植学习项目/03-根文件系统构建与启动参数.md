# 根文件系统构建与启动参数

## 导读

### 本章定位

这一章看 Linux 内核起来以后，如何找到根文件系统、挂载根文件系统，并最终进入 `/sbin/init`。

### 核心对象

- `bootargs`
  - 传给内核的启动参数
- `root=`
  - 根文件系统位置
- `init=`
  - 首个用户态程序
- `initramfs / initrd`
  - 内存中的根文件系统形式
- `rootfs`
  - 磁盘或 flash 上的根文件系统内容

### 关键函数

- `parse_early_param()`
- `parse_args()`
- `prepare_namespace()`
- `mount_root()`
- `run_init_process()`

### 主流程

接收 `bootargs` -> 解析 `root=`、`console=`、`init=` -> 识别块设备或内存根文件系统 -> 挂载根文件系统 -> 执行 `init`。

## 1. 根文件系统层真正解决什么

- 提供 `/sbin/init`
- 提供 `/dev`、`/proc`、`/sys`
- 提供 shell、动态库、基础命令
- 提供业务程序运行所需环境

## 2. 常见根文件系统形式

- `initramfs`
  - 打包进内核或由引导程序加载到内存
- `NFS root`
  - 通过网络挂载
- `eMMC/SD/NAND` 上的 ext4、squashfs、ubifs、jffs2

## 3. 最容易出的问题

- `VFS: Cannot open root device`
- 找不到 `/sbin/init`
- 动态库缺失
- 设备节点不完整
- `console=` 错误导致系统起来但没有输出

## 4. 关键参数

- `console=`
- `root=`
- `rootfstype=`
- `rootwait`
- `init=`
- `ubi.mtd=` / `ubi.block=` / `mtdparts=`

## 5. 一句话总结

根文件系统层决定“内核起来以后，系统到底能不能进入真正可交互的用户态”。
