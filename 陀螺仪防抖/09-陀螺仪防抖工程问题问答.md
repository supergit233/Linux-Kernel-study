# 陀螺仪防抖工程问题问答

## 学习目标

- 按现象把问题定位到 gyro、video、timestamp、标定、算法、warp 或性能层
- 避免一看到效果不好就直接怀疑滤波算法
- 建立调试日志和验证实验的固定顺序

## 导读

### 本章定位

这一章是工程排查入口。陀螺仪防抖效果问题通常是跨层问题，必须先分层，再决定看驱动、看时间戳、看标定还是看算法。

### 核心对象

- gyro sample
- frame timestamp
- calibration model
- correction transform
- warp output

### 关键函数

当前先按调试入口记录：

- gyro sample dump
- frame timestamp dump
- calibration config dump
- stabilization transform dump
- warp output comparison

### 主流程

```text
观察现象                         // 抖、漂、反向、黑边、延迟、撕裂
-> 判断层级                       // gyro/video/time/calibration/algorithm/warp/performance
-> 打印关键对象                    // 样本、时间戳、标定、补偿量、输出帧
-> 做最小实验                      // 静止、单轴转动、固定帧率离线回放
-> 修正对应层                      // 不跨层盲改参数
```

## 1. 现象到层级判断表

| 现象 | 优先怀疑层级 | 第一检查点 |
| --- | --- | --- |
| 静止画面慢慢漂 | gyro bias | 静止段 `gx/gy/gz` 均值是否接近 0 |
| 快速转动时补偿慢半拍 | 时间同步 | `time_offset`、frame timestamp 来源 |
| 补偿方向反了 | 坐标系/外参 | gyro 轴方向、`R_gyro_to_cam` |
| 只在某个方向防抖差 | 轴映射/外参 | 单轴转动测试 |
| 边缘黑边明显 | crop/warp | crop margin 是否不足 |
| 画面像果冻一样扭曲 | rolling shutter | 是否只用了整帧 transform |
| 偶发大跳变 | gyro 丢样或 timestamp 跳变 | sample seq、timestamp 单调性 |
| 开启防抖后帧率下降 | 性能链路 | warp 耗时、buffer 拷贝、线程优先级 |
| 视频链路正常但防抖无效果 | 算法/接入 | transform 是否真的应用到输出帧 |

## 2. Q：为什么防抖效果不好不能先调滤波参数

因为滤波只处理轨迹平滑，不能修复下面这些基础错误：

- gyro 单位错
- 坐标轴错
- timestamp 延迟错
- frame timestamp 不是曝光时间
- rolling shutter 用了整帧模型
- crop margin 不够

正确顺序是：

```text
先验证 gyro 数据                    // 单位、零偏、采样率、timestamp
-> 再验证 video timestamp           // 帧率、SOF/EOF/done 含义
-> 再验证坐标系和外参                // 单轴转动看方向
-> 再调算法平滑参数                  // 最后才改滤波窗口和补偿强度
```

## 3. Q：如何快速判断补偿方向是否反了

做单轴小角度实验：

```text
相机向右缓慢转动                    // 只做一个方向的明确运动
-> 记录 gyro 三轴曲线                // 看主要响应轴和符号
-> 输出 correction 曲线              // 看补偿方向是否与原始运动相反
-> 观察画面边缘运动                  // 判断是否越补越抖
```

如果某一轴反了，不要直接改算法整体符号，要先确认：

- gyro 芯片坐标
- 板级安装方向
- camera 坐标系定义
- 图像 x/y 坐标方向

## 4. Q：如何判断时间延迟问题

典型特征：

- 小抖动有效，大幅快速运动差
- 转动停止后画面还继续补偿一小段
- 调整平滑参数不能根治相位滞后

调试方法：

```text
固定一段视频和 gyro 数据             // 保证输入不变
-> 离线扫描不同 time_offset          // 例如 -50ms 到 +50ms
-> 计算或观察稳定效果                // 找到效果最好的 offset
-> 回到实时系统验证                  // 检查实时链路是否有同样延迟
```

## 5. Q：怎么区分 gyro 问题和视频 timestamp 问题

先分别画曲线：

- gyro timestamp 间隔是否稳定
- frame timestamp 间隔是否稳定
- gyro 时间范围是否覆盖视频帧
- 快速运动峰值是否出现在视频运动之前或之后

如果 gyro 采样间隔抖动大，先查 gyro 链路；如果 gyro 稳定但 frame timestamp 含义不清，先查视频链路。

## 6. Q：为什么离线有效，实时效果差

常见原因：

- 离线用估算时间戳，实时用 buffer done timestamp
- 实时 gyro FIFO 引入延迟
- 实时 warp 耗时导致帧延迟
- 实时线程抢占导致样本丢失
- 离线用了未来帧平滑，实时不能使用同样窗口

## 回答的问题

- 防抖漂移先看哪里
- 方向反了如何验证
- 时间同步怎么调
- 离线和实时为什么效果不同
- 黑边、撕裂、果冻效应分别对应哪些层级
