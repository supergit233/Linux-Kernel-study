# 防抖算法主流程

## 导读

### 本章定位

这一章讲陀螺仪视频防抖算法的主流程。目标不是先追求复杂滤波器，而是建立从 gyro 角速度到图像 warp 的完整闭环。`accel` 和 `temp` 在这里主要作为标定、姿态检查和温漂排查输入，不是第一版图像补偿的主路径。

### 核心对象

- calibrated gyro sample
- rotation estimate
- raw camera path
- smooth camera path
- correction transform
- crop margin

### 关键函数

当前先按算法模块记录：

- gyro bias correction
- angular velocity integration
- rotation representation
- path smoothing
- transform generation
- warp/crop

### 主流程

```text
读取校准后的 gyro 样本                 // 输入已经去零偏、统一单位和坐标系
-> 对时间窗口积分                       // 得到当前帧或当前行的相机旋转
-> 累积 raw camera path                 // 形成原始抖动轨迹
-> 平滑 raw path                        // 生成目标稳定轨迹
-> 计算 correction                       // 原始轨迹和目标轨迹之间的差
-> 生成图像 transform                    // 转成 homography、mesh 或行变换
-> warp + crop                           // 应用补偿并裁剪无效边缘
```

因此本章后面默认讨论 `gyro`。如果后续要做更完整的姿态融合，才会把 `accel` 引入低频姿态约束；如果要做长时间温漂补偿，才会把 `temp` 引入 bias 模型。

## 这一章按什么逻辑展开

本章按算法数据流展开：先估计运动，再平滑轨迹，再生成补偿。这样可以把“运动估计”和“图像变换”分开看。

## 1. 角速度积分

gyro 输出的是角速度，不是角度。

角速度说的是“现在转得多快”，角度说的是“已经转了多少”。防抖最终要知道画面应该反向补多少角度，所以需要把一段时间内的角速度累加起来。

```text
angular velocity w(t)                  // gx/gy/gz，单位通常转成 rad/s
-> integrate over dt                   // 每小段都做“转得多快 × 持续多久”
-> delta rotation                      // 得到这一小段时间转了多少
-> accumulated orientation             // 把很多小段加起来，形成相机姿态轨迹
```

实现上可以先从小角度近似开始，再逐步过渡到旋转矩阵或四元数。

## 2. 原始轨迹和目标轨迹

防抖不是简单把所有运动抵消掉，否则正常转动也会被锁死。更合理的模型是：

```text
raw camera path                       // gyro 看到的真实运动：手抖 + 主动转动
-> smooth camera path                 // 希望观众看到的运动：保留慢慢转动，去掉小抖动
-> correction = smooth - raw          // 两条轨迹差多少，就反向补多少
```

可以这样想：如果用户是慢慢向右转镜头，这个运动应该保留；如果手在高频小抖，这部分应该被抵消。`smooth camera path` 就是算法认为“看起来舒服”的目标路线。

平滑窗口越大：

- 防抖越稳
- 画面延迟越大
- 需要更大的 crop margin

## 3. 补偿变换

纯旋转近似下，可以把相机旋转转换成图像 homography：

`homography` 先不用想得太数学，它可以理解成“整张图像按照一个统一规则挪动、旋转、拉伸”的变换矩阵。对远处场景、相机主要是旋转的情况，用它做第一版防抖比较合适。

```text
rotation correction R                 // 相机坐标系里的补偿旋转
-> camera intrinsic K                 // 相机内参矩阵
-> H = K * R * K^-1                   // 得到图像平面变换
-> inverse warp                       // 反过来采样原图，生成稳定后的图像
```

这里的 `inverse warp` 可以理解成：输出图像上的每个像素，反过来去原图里找它应该来自哪里。这样比“把原图像素往外推”更容易避免空洞。

如果只做第一版实验，可以先实现：

- 小角度平移近似
- 整帧 rotation/homography
- 固定 crop

后续再扩展：

- rolling shutter per-line warp
- 畸变校正联合处理
- mesh warp
- 动态 crop

## 4. crop margin

电子防抖一定需要预留边界。画面补偿后，边缘会出现无效区域，需要裁剪。

`crop margin` 就是提前多留的一圈画面。比如最终要输出 1920x1080，可以让 sensor/ISP 先给更大一点的画面，防抖时画面怎么挪都尽量不露黑边，最后再裁回目标大小。

```text
原始图像尺寸                         // sensor 或 ISP 输出分辨率
-> 预留 crop margin                  // 比最终输出大一圈
-> warp 后裁剪                       // 去掉补偿造成的黑边
-> 输出稳定画面                      // 分辨率通常小于原始采集区域
```

crop margin 越大，可补偿幅度越大，但视角损失也越大。

## 5. rolling shutter 的算法边界

第一版可以先忽略 rolling shutter，把一帧当成同一时刻曝光。真实手机/IPC sensor 常常是 rolling shutter，此时每一行对应不同姿态。

`rolling shutter` 可以先理解成“画面不是整张同时拍下来的，而是一行一行扫出来的”。如果相机在扫的过程中还在动，画面上方和下方对应的相机姿态就不一样。整帧只用一个补偿矩阵时，可能中心稳定了，边缘或上下区域还像果冻一样扭。

```text
frame-level transform                // 简单，适合第一版
-> line-level transform              // 更真实，适合 rolling shutter
-> mesh warp                         // 工程上常用来表达复杂形变
```

## 本章边界

本章只给算法主流程，不绑定具体数学库和源码。第一版目标是能离线跑通，而不是一次性实现工业级 EIS。

## 一句话总结

陀螺仪防抖算法的本质是：用 gyro 估计原始相机运动，再生成一条更平滑的目标运动，并把两者差值转换成图像补偿。
