# 标定：内参、外参、零偏与坐标系

## 导读

### 本章定位

这一章讲防抖前必须确定的标定参数。时间戳解决“什么时候”，标定解决“哪个方向、多少量、映射到图像哪里”。

### 核心对象

- gyro bias
- gyro scale
- accel bias / accel scale
- temperature drift
- axis mapping
- camera intrinsic
- lens distortion
- gyro-camera extrinsic
- time offset

### 关键函数

当前先按标定阶段记录：

- 静止零偏估计
- scale/axis 校准
- camera intrinsic calibration
- gyro-camera extrinsic calibration
- delay calibration

### 主流程

```text
静止采集 accel/gyro/temp             // 同时观察重力、零偏和温度
-> 估计 gyro bias                    // 静止时角速度均值就是要减掉的假速度
-> 检查单位和量程                    // 确认 accel/gyro 换算比例正确
-> 确认坐标轴方向                    // 建立设备坐标系、gyro 坐标系和 camera 坐标系关系
-> 标定相机内参                      // 得到焦距、主点、畸变
-> 标定 gyro-camera 外参             // 得到 gyro 到 camera 的旋转关系
-> 标定时间延迟                      // 修正 gyro 和 frame 的时间偏移
```

## 这一章按什么逻辑展开

本章按“先修 gyro 自身，再修 camera，再修二者关系”的顺序展开。因为任何一个底层标定错误，都会被后面的算法放大成画面反向、漂移或拉扯。

## 1. gyro bias

零偏是陀螺仪静止时仍然输出的角速度。

换成人话说：设备明明没动，gyro 还说“我在慢慢转”，这个假的转动速度就是 `bias`。防抖算法如果不先减掉它，就会以为相机一直在微微旋转。

```text
设备静止放置                         // 保证真实角速度接近 0
-> 连续采集 accel/gyro/temp            // 同时保存 ax/ay/az、gx/gy/gz 和温度
-> 求均值                             // 得到 bias
-> 后续每个样本减 bias                 // 减少积分漂移
```

常见现象：

- 静止画面慢慢旋转
- 长时间视频出现累积漂移
- 不同温度下防抖效果变化

静止采集时不要只看 `gyro`。`accel` 可以告诉当前板子大概怎么放置，`temp` 可以告诉这段零偏是在什么温度下测到的。以后如果热机后漂移变大，就能回头比较“温度是否变了、gyro bias 是否跟着变了”。

## 2. scale 和单位

角速度单位必须统一。算法里通常更适合使用 `rad/s`。加速度单位也要明确，常见是 `g` 或 `m/s^2`。

`scale` 是“原始数字到真实物理量”的换算比例。gyro 芯片读出来的可能只是一个原始数，比如 `1000`，它不直接等于 `1000 deg/s` 或 `1000 rad/s`。accel 也是一样，原始数必须按量程换算后才知道是 `0.5g`、`1g` 还是别的值。

```text
raw value                          // 芯片寄存器或驱动给出的原始数字
× scale                            // 换算比例
= 真实物理量                        // gyro 是角速度，accel 是加速度
```

“修正比例误差”可以这样理解：真实转了 10 度，但算法根据当前数据只算出 8 度，说明整个比例偏小；如果算出 12 度，说明比例偏大。`scale` 就是把这个比例关系校正回来。

检查方法：

- 查芯片 datasheet 的 LSB 与 dps 对应关系
- 查驱动是否已经应用 IIO scale
- 静止时检查 accel 三轴合成值是否接近 `1g`
- 做一次已知角度旋转，积分结果是否接近真实角度

如果 `deg/s` 被当成 `rad/s`，补偿量会放大约 57.3 倍。

## 3. accel 和 temp 的标定价值

`accel` 和 `temp` 不一定直接进入第一版视频防抖补偿，但它们会让调试少走很多弯路。

| 数据 | 标定里主要回答的问题 | 怎么用 |
| --- | --- | --- |
| `accel` | 板子静止时，重力主要落在哪个轴上 | 判断芯片安装方向、轴向符号、加速度 scale 是否基本正确 |
| `gyro` | 板子静止时，角速度平均值是多少 | 得到每个轴的 `gyro_bias`，后续每个样本先减掉 |
| `temp` | 当前 bias 是在什么温度下测到的 | 长时间运行时记录温度和 bias 的关系，必要时做温度补偿 |

例如静止放置时，`accel` 的合成值如果接近 `1g`，说明加速度换算大体可信；如果某个 `gyro` 轴静止时长期在 `1 dps` 附近，说明这个轴存在约 `1 dps` 的零偏；如果温度升高后这个零偏也变了，就要考虑温漂。

## 4. axis mapping

陀螺仪芯片坐标轴、板子坐标轴和相机图像坐标轴通常不完全一致。

`axis mapping` 就是回答一个很朴素的问题：gyro 的 X/Y/Z 三个方向，到底对应画面里的哪个方向？

例如板子设计时 gyro 芯片可能旋转了 90 度贴上去，那么 gyro 的 X 轴就不一定是相机画面里的水平轴。这个映射不弄清楚，算法就可能把“向右晃”误认为“向上晃”。

```text
gyro chip axis                       // 芯片封装定义的 X/Y/Z
-> board mounting axis               // PCB 实际安装方向
-> camera optical axis               // 相机坐标系
-> image x/y axis                    // 图像平面坐标
```

调试方法：

- 静止时先看 `accel` 哪个轴接近 `1g` 或 `-1g`
- 绕相机水平轴转动，观察主要变化的是哪个 gyro 轴
- 绕垂直轴转动，观察符号是否符合预期
- 做小角度正反方向测试，确认补偿不是反向

## 5. camera intrinsic

相机内参把相机坐标系中的运动映射到图像平面。

`camera intrinsic` 可以先理解成“相机自己的成像规则”。同样是相机转 1 度，广角镜头和长焦镜头在画面上移动的像素数不一样；画面中心在哪里、焦距是多少，都会影响补偿量。

常见参数：

- `fx/fy`
- `cx/cy`
- distortion coefficients

简化旋转防抖常用关系：

```text
gyro rotation R                     // 相机姿态变化
-> camera intrinsic K               // 像素坐标和相机坐标之间的映射
-> H = K * R * K^-1                 // 纯旋转近似下的图像 homography
-> warp frame                       // 用 H 或其逆变换补偿画面
```

## 6. gyro-camera extrinsic

gyro 测到的是 gyro 坐标系里的旋转，相机图像需要的是 camera 坐标系里的旋转。

`extrinsic` 可以先理解成“gyro 和 camera 两个零件之间怎么摆放”。gyro 负责感知运动，但最终要稳定的是相机画面。如果两者安装方向不一样，就必须把 gyro 的运动翻译成相机坐标系里的运动。

```text
gyro angular velocity               // gyro 坐标系
-> R_gyro_to_cam                    // 外参旋转
-> camera angular velocity          // camera 坐标系
-> image transform                  // 映射到画面补偿
```

如果外参不对，常见表现是：

- 水平转动引起垂直补偿
- 画面补偿方向不稳定
- 某些方向有效，某些方向更差

## 7. time offset 也属于标定参数

时间延迟可以放在 [[04-时间戳同步机制]] 里分析，但工程上它也常作为标定参数保存。

这里的“标定 time offset”就是测出 gyro 和 video 固定差了几毫秒。比如同一次手抖，gyro 日志里 100ms 出现峰值，视频画面里 118ms 才看到对应运动，那就说明两条链路之间大约有 18ms 的差值需要补偿。

```text
gyro time                          // 陀螺仪采样时刻
-> time_offset                     // 标定出的固定延迟
-> aligned frame time              // 用于查找或积分 gyro 样本
```

## 本章边界

本章只讲标定参数的意义和工程影响，不展开完整标定工具实现。离线实验章节会给出更适合快速验证的流程。

## 一句话总结

防抖算法不是拿原始 gyro 直接算补偿，而是拿经过零偏、单位、坐标系、相机内参、外参和时间延迟修正后的运动模型来算补偿；`accel` 用来辅助确认姿态和安装方向，`temp` 用来解释零偏随温度变化。
