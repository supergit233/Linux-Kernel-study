# 陀螺仪防抖学习达标检查清单

## 学习目标

- 明确学完陀螺仪防抖项目后应该达到什么效果
- 把知识拆成采集、时间、标定、算法、实时和调试几个可检查维度
- 提供复盘时可对照的问题清单

## 导读

### 本章定位

这一章是项目收口章，不继续展开新技术细节，而是判断是否已经具备独立阅读、实验和排查陀螺仪防抖问题的能力。

### 核心对象

- IMU sample
- gyro sample
- video frame
- timestamp
- calibration model
- stabilization transform

### 关键函数

当前按入口类型检查：

- gyro 采集入口
- V4L2 采集入口
- 时间同步入口
- 标定入口
- 防抖算法入口
- warp 输出入口

### 主流程

```text
IMU 采样
-> 区分 accel/gyro/temp
-> video frame
-> 时间同步
-> 标定修正
-> 姿态积分
-> 轨迹平滑
-> 图像补偿
-> 实时输出
```

## 1. 学完后应该达到的效果

- 能画出 IMU/gyro 到稳定视频的完整数据流
- 能用人话解释 `ax/ay/az`、`gx/gy/gz`、`temp`、`scale`、`bias`、`time_offset`、`crop_margin` 这些字段是干什么的
- 能区分 accel、gyro、temp 在防抖项目里的不同角色
- 能解释为什么 timestamp 是防抖成败关键
- 能区分 gyro bias、axis mapping、camera intrinsic、gyro-camera extrinsic、time offset
- 能用离线实验验证最小防抖闭环
- 能把实时问题拆成采集、算法、warp、性能和输出链路
- 能根据现象判断优先排查哪一层

## 2. 必须知道的内容清单

### 2.1 IMU/gyro 采集清单

- [ ] IMU/gyro 数据从 I2C/SPI 到驱动 probe 的入口是什么
- [ ] IMU/gyro 使用 IIO、input 还是私有接口暴露数据
- [ ] 一次 IMU 输出里是否包含 `accel/gyro/temp`
- [ ] `ax/ay/az` 是加速度，不是速度或位移；静止时主要反映重力方向
- [ ] `gx/gy/gz` 分别表示什么，和画面抖动有什么关系
- [ ] `temp` 为什么需要记录，它和 gyro 零偏温漂有什么关系
- [ ] `scale` 为什么不是一句“修正比例误差”就够了，它到底把什么换成什么
- [ ] accel 和 gyro 的采样率、量程、单位和 scale 是否明确
- [ ] 每个样本是否有 timestamp
- [ ] FIFO/IRQ/trigger 如何影响延迟和丢样
- [ ] 静止零偏如何估计
- [ ] 静止时 accel 合成值是否接近 `1g`

### 2.2 视频采集清单

- [ ] 视频链路包含 sensor、CSI、ISP、V4L2 哪些节点
- [ ] V4L2 buffer timestamp 表示 SOF、EOF、DMA done 还是其他时刻
- [ ] frame sequence 是否连续
- [ ] 帧率是否稳定
- [ ] 是否有 exposure、frame duration、metadata
- [ ] sensor 是 global shutter 还是 rolling shutter

### 2.3 时间同步清单

- [ ] gyro timestamp 和 frame timestamp 是否同源
- [ ] 如果不同源，如何换算
- [ ] time offset 如何估计和验证
- [ ] frame timestamp 如何转换到曝光中心
- [ ] gyro 样本如何插值或积分到帧时刻
- [ ] 丢样、丢帧时如何处理

### 2.4 标定清单

- [ ] gyro bias 如何估计
- [ ] gyro 单位是否统一成算法需要的单位
- [ ] gyro 轴和相机轴如何对应
- [ ] 相机内参和畸变参数是否可用
- [ ] gyro-camera 外参如何表示
- [ ] time offset 是否作为标定参数保存

### 2.5 算法清单

- [ ] 角速度如何积分成姿态变化
- [ ] raw path 和 smooth path 有什么区别
- [ ] correction transform 如何生成
- [ ] crop margin 如何决定最大补偿幅度
- [ ] rolling shutter 场景为什么可能需要逐行补偿
- [ ] 什么时候应该跳过防抖或重置状态

### 2.6 工程排查清单

- [ ] 静止漂移时优先查 gyro bias
- [ ] 补偿慢半拍时优先查 time offset
- [ ] 补偿反向时优先查坐标轴和外参
- [ ] 黑边明显时优先查 crop margin 和 warp
- [ ] 果冻效应明显时优先查 rolling shutter 模型
- [ ] 实时帧率下降时优先查 warp 耗时、拷贝和线程调度

## 3. 能独立回答的问题

- 陀螺仪防抖和纯图像防抖有什么区别
- `OIS`、`EIS`、`Gyro EIS` 的边界是什么
- 为什么 gyro sample 必须带 timestamp
- 为什么 MPU6050 会同时输出 accel、gyro、temp
- 为什么视频防抖主输入是 gyro，但 accel 和 temp 仍然要保存
- 为什么静止时 accel 主要用来检查重力和安装方向
- 为什么温度变化可能导致 gyro bias 改变
- 为什么 V4L2 buffer timestamp 不一定是曝光时间
- 为什么陀螺仪零偏会导致画面漂移
- 为什么坐标轴错会导致补偿反向
- 为什么 crop margin 决定防抖上限
- 为什么 rolling shutter 会让整帧 homography 不够用
- 为什么离线防抖有效不代表实时一定有效
- 一个防抖现象如何映射到采集、同步、标定、算法或性能层

## 4. 不达标信号

- 只会说“用陀螺仪算旋转”，但说不清 timestamp 如何对齐
- 能读 MPU6050 值，但分不清 `accel`、`gyro`、`temp` 分别用来做什么
- 能读 gyro 值，但不知道单位、scale、bias 和坐标轴
- 看到 `val accel=(...)g`，不知道静止时应该用它检查重力方向和 `1g` 合成值
- 看到 `temp=(...)C`，不知道它和长期漂移、零偏温漂有什么关系
- 能背出 `scale`、`bias`、`外参`，但不能解释它们错了会让画面出现什么现象
- 能拿到视频帧，但不知道 timestamp 是 SOF、EOF 还是 done
- 效果不好时直接调滤波参数，不先查时间和坐标系
- 不知道 `raw path` 和 `smooth path` 的区别
- 不知道 crop margin 为什么会造成视角损失
- 离线和实时效果不一致时无法分层排查

## 5. 达标复盘方式

1. 先不看笔记画出 IMU/gyro 到稳定视频的完整流程
2. 用 [[01-陀螺仪防抖核心对象]] 检查对象字段是否清楚
3. 用 [[02-陀螺仪数据采集链路]] 和 [[03-视频采集与V4L2链路]] 检查输入链路
4. 用 [[04-时间戳同步机制]] 检查时间轴是否讲得清
5. 用 [[05-标定：内参、外参、零偏与坐标系]] 检查坐标和参数
6. 用 [[06-防抖算法主流程]] 检查算法闭环
7. 用 [[07-离线OpenCV实验]] 做最小验证
8. 用 [[08-嵌入式实时落地]] 和 [[09-陀螺仪防抖工程问题问答]] 做工程排查

## 6. 一句话标准

真正达标的标准是：看到一个防抖现象时，能先判断它卡在 `gyro 采集 -> video timestamp -> 时间同步 -> 标定 -> 算法 -> warp/crop -> 实时性能` 的哪一段，再决定看哪份源码、打哪条日志、做哪个实验。
