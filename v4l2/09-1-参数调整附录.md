完整路径可以按 **“用户态 ioctl → V4L2 control 框架 → sensor s_ctrl → I2C 写寄存器”** 来理解。

以设置曝光为例：

```c
struct v4l2_control ctrl;

ctrl.id = V4L2_CID_EXPOSURE;
ctrl.value = 1000;

ioctl(fd, VIDIOC_S_CTRL, &ctrl);
```

这里的 `fd` 可能是 `/dev/video0`，也可能是 `/dev/v4l-subdevX`，看平台怎么暴露 control。

---

## 1. 总体路径

```text
用户态 APP
  ↓
ioctl(fd, VIDIOC_S_CTRL, &ctrl)
  ↓
系统调用进入内核
  ↓
/dev/videoX 的 fops->unlocked_ioctl
  ↓
v4l2_ioctl()
  ↓
video_ioctl2()
  ↓
__video_do_ioctl()
  ↓
V4L2 control 框架
  ↓
找到 V4L2_CID_EXPOSURE
  ↓
检查范围、步进、权限
  ↓
调用 sensor 驱动的 s_ctrl()
  ↓
sensor_set_exposure()
  ↓
I2C 写 sensor 曝光寄存器
  ↓
sensor 曝光时间改变
```

最核心的一句话是：

> 用户态通过 `VIDIOC_S_CTRL` 设置 `V4L2_CID_EXPOSURE`，V4L2 core 找到对应 control 后，调用 sensor 驱动注册的 `s_ctrl` 回调，最后 sensor 驱动通过 I2C 写曝光寄存器。

---

## 2. 用户态发起设置曝光

用户态代码一般是：

```c
struct v4l2_control ctrl;

memset(&ctrl, 0, sizeof(ctrl));
ctrl.id = V4L2_CID_EXPOSURE;
ctrl.value = 1000;

ioctl(fd, VIDIOC_S_CTRL, &ctrl);
```

含义是：

```text
我要设置曝光参数，control ID 是 V4L2_CID_EXPOSURE，值是 1000。
```

注意，这里的 `1000` 通常不是直接表示 `1000 ms`。

在 sensor 驱动里，曝光值一般表示：

```text
曝光行数，也叫 coarse integration time
```

比如：

```text
一行时间 = 10 us
曝光值 = 1000 行
实际曝光时间 = 1000 × 10 us = 10 ms
```

---

## 3. ioctl 先进入 video device

如果用户打开的是：

```c
fd = open("/dev/video0", O_RDWR);
```

那么 `ioctl()` 首先进入 `/dev/video0` 对应的 `video_device`。

路径类似：

```text
ioctl(fd, VIDIOC_S_CTRL)
  ↓
v4l2_fops.unlocked_ioctl
  ↓
v4l2_ioctl()
  ↓
video_ioctl2()
  ↓
__video_do_ioctl()
```

这里处理的是 **用户态到 V4L2 core 的通用 ioctl 流程**。

也就是说，此时还没有直接到 sensor。

---

## 4. V4L2 core 识别这是 control 操作

因为命令是：

```c
VIDIOC_S_CTRL
```

所以 V4L2 core 知道这是在设置一个 control。

然后它会根据：

```c
ctrl.id = V4L2_CID_EXPOSURE;
```

去当前设备的 control handler 里面找这个 control。

也就是找：

```text
这个设备有没有 V4L2_CID_EXPOSURE 这个控制项？
它的最小值是多少？
最大值是多少？
步进是多少？
当前值是多少？
由哪个驱动处理？
```

sensor 驱动里通常会注册：

```c
v4l2_ctrl_new_std(handler,
                  &sensor_ctrl_ops,
                  V4L2_CID_EXPOSURE,
                  min,
                  max,
                  step,
                  default_value);
```

这表示 sensor 支持曝光控制。

---

## 5. V4L2 control 框架做检查

在真正调用 sensor 驱动前，V4L2 control 框架会先检查：

```text
control ID 是否存在
值是否在 min/max 范围内
是否满足 step
这个 control 当前是否可写
设备状态是否允许设置
```

比如 sensor 注册曝光范围是：

```text
min = 1
max = 2000
step = 1
default = 500
```

用户设置：

```text
value = 1000
```

合法，继续。

如果用户设置：

```text
value = 999999
```

可能会被拒绝，或者被驱动 clamp 到最大值，具体看驱动实现。

---

## 6. 调用 sensor 的 s_ctrl

control 合法后，会调用 sensor 驱动注册的回调。

sensor 里通常有：

```c
static const struct v4l2_ctrl_ops sensor_ctrl_ops = {
    .s_ctrl = sensor_set_ctrl,
};
```

然后进入：

```c
static int sensor_set_ctrl(struct v4l2_ctrl *ctrl)
{
    struct sensor *sensor =
        container_of(ctrl->handler, struct sensor, ctrl_handler);

    switch (ctrl->id) {
    case V4L2_CID_EXPOSURE:
        return sensor_set_exposure(sensor, ctrl->val);

    case V4L2_CID_ANALOGUE_GAIN:
        return sensor_set_gain(sensor, ctrl->val);

    case V4L2_CID_VBLANK:
        return sensor_set_vblank(sensor, ctrl->val);

    default:
        return -EINVAL;
    }
}
```

这里就真正进入 sensor 驱动了。

所以：

```text
VIDIOC_S_CTRL
    ↓
V4L2 control framework
    ↓
sensor_ctrl_ops.s_ctrl()
```

---

## 7. sensor_set_exposure() 把曝光值转换成寄存器值

sensor 驱动拿到的可能是：

```text
ctrl->val = 1000
```

这一般表示曝光行数。

然后驱动会把它写进 sensor 的曝光寄存器。

例如很多 sensor 曝光寄存器可能是两个字节：

```c
#define SENSOR_REG_EXPOSURE_H  0x0202
#define SENSOR_REG_EXPOSURE_L  0x0203
```

驱动可能这样写：

```c
static int sensor_set_exposure(struct sensor *sensor, u32 exposure)
{
    int ret;

    ret = sensor_write_reg(sensor, 0x0202, (exposure >> 8) & 0xff);
    if (ret)
        return ret;

    ret = sensor_write_reg(sensor, 0x0203, exposure & 0xff);
    if (ret)
        return ret;

    return 0;
}
```

所以：

```text
用户态设置 1000
    ↓
驱动拆成高低字节
    ↓
写 0x0202
    ↓
写 0x0203
```

不同 sensor 的寄存器地址不一样，但逻辑类似。

---

## 8. 最后通过 I2C 写入 sensor

`sensor_write_reg()` 底层通常会走 I2C。

例如：

```c
static int sensor_write_reg(struct sensor *sensor, u16 reg, u8 val)
{
    u8 buf[3];

    buf[0] = reg >> 8;
    buf[1] = reg & 0xff;
    buf[2] = val;

    return i2c_master_send(sensor->client, buf, 3);
}
```

继续往下就是 Linux I2C 子系统：

```text
sensor_write_reg()
  ↓
i2c_master_send() / i2c_transfer()
  ↓
I2C core
  ↓
I2C adapter driver
  ↓
SoC I2C controller
  ↓
SCL / SDA
  ↓
sensor 芯片寄存器
```

最后 sensor 内部曝光寄存器被改掉。

---

## 9. 如果 sensor 没有上电怎么办？

这个也很重要。

有些驱动会这样处理：

```c
if (!pm_runtime_get_if_in_use(dev))
    return 0;
```

意思是：

```text
如果 sensor 当前没上电，就先不真正写寄存器，只保存 control 的值。
```

等到后面 `s_stream(1)` 时，再统一应用：

```c
__v4l2_ctrl_handler_setup(&sensor->ctrl_handler);
```

也就是说，有两种情况：

### 情况一：sensor 正在工作

```text
VIDIOC_S_CTRL 设置曝光
  ↓
立即进入 s_ctrl
  ↓
立即 I2C 写曝光寄存器
  ↓
后续帧曝光改变
```

### 情况二：sensor 还没 stream on

```text
VIDIOC_S_CTRL 设置曝光
  ↓
V4L2 control 框架保存新值
  ↓
sensor 暂时不写寄存器
  ↓
VIDIOC_STREAMON
  ↓
sensor_s_stream(1)
  ↓
写 mode 表
  ↓
__v4l2_ctrl_handler_setup()
  ↓
把曝光、增益等 control 写入 sensor
```

所以曝光 control 不一定每次都立刻写硬件，取决于 sensor 当前是否上电、是否正在 streaming，以及驱动怎么写。

---

## 10. 如果用户打开的是 /dev/v4l-subdevX

有些平台会暴露 subdev 节点：

```text
/dev/v4l-subdev0
/dev/v4l-subdev1
```

如果用户直接对 sensor subdev 节点调用：

```c
ioctl(subdev_fd, VIDIOC_S_CTRL, &ctrl);
```

路径会更直接：

```text
用户态
  ↓
/dev/v4l-subdevX
  ↓
subdev ioctl
  ↓
sensor 的 control handler
  ↓
sensor s_ctrl
  ↓
I2C 写曝光寄存器
```

而如果用户对 `/dev/video0` 设置曝光，中间通常还会经过 video device/capture driver，然后找到或转发到 sensor control。

---

## 11. 和 set_fmt / s_stream 的关系

曝光设置和格式设置不是一条路。

```text
设置分辨率：
VIDIOC_S_FMT
  ↓
video ioctl_ops
  ↓
subdev pad_ops.set_fmt
  ↓
选择 sensor mode

开始采集：
VIDIOC_STREAMON
  ↓
vb2 / capture driver
  ↓
subdev video_ops.s_stream(1)
  ↓
sensor 写 mode 表和 streaming on

设置曝光：
VIDIOC_S_CTRL
  ↓
V4L2 control framework
  ↓
sensor ctrl_ops.s_ctrl
  ↓
I2C 写曝光寄存器
```

所以：

```text
分辨率走 set_fmt
启停流走 s_stream
曝光增益走 s_ctrl
```

## 12. 完整面试版回答

可以这样说：

```text
用户态设置曝光时，一般通过 VIDIOC_S_CTRL 或 VIDIOC_S_EXT_CTRLS，
control id 是 V4L2_CID_EXPOSURE。

ioctl 先从用户态进入 /dev/videoX 的 file_operations，
经过 v4l2_ioctl、video_ioctl2、__video_do_ioctl 后，
V4L2 core 识别这是 control 操作。

然后 V4L2 control 框架根据 V4L2_CID_EXPOSURE 找到对应的 control，
检查设置值是否在 min/max/step 范围内。
检查通过后，会调用 sensor 驱动注册的 v4l2_ctrl_ops.s_ctrl 回调。

sensor 的 s_ctrl 里根据 ctrl->id 判断是曝光控制，
然后把 ctrl->val 转换成 sensor 需要的曝光寄存器值，
通常是曝光行数 coarse integration time，
再通过 I2C 写入 sensor 的曝光寄存器。

如果 sensor 当前没有上电或没有 streaming，
驱动可能只是保存 control 值，等到 s_stream(1) 时再通过
__v4l2_ctrl_handler_setup() 把曝光、增益等参数统一写进 sensor。

所以完整路径是：
用户态 VIDIOC_S_CTRL →
V4L2 ioctl →
V4L2 control framework →
sensor s_ctrl →
I2C 写 sensor exposure register。
```

最短版：

```text
VIDIOC_S_CTRL 是用户态入口；
V4L2_CID_EXPOSURE 是要设置的 control；
V4L2 core 负责查找和校验 control；
sensor 的 s_ctrl 负责把曝光值转成寄存器值；
I2C 负责真正写进 sensor。
```
